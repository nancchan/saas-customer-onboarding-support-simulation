I created this project while learning the core skills used by SaaS Support Engineers, Technical Support Engineers, and Implementation Specialists.

The project simulates a customer onboarding process, following customer data from a raw CSV file through data cleaning, SQL validation, API testing with Postman, and Salesforce CRM mapping. It focuses on the troubleshooting and investigative workflow used to identify and resolve common onboarding issues, such as invalid customer data, duplicate records, and CRM synchronization failures.

Rather than building a production application, this project serves as a hands-on portfolio demonstrating my understanding of customer data flows, SaaS support processes, and cross-system troubleshooting.
